Tomcat笔记

2022-05-12

JavaWeb的概念

request/response

Web资源的分类

静态资源：html/css/js/txt/mp4/jpg

动态资源：jsp页面/Servlet程序

常用的服务器

Tomcat/Jboss/GlassFish/Resin/WebLogic

Tomcat的使用

  1. 服务器的安装

    解压压缩包即可

  2. Tomcat的目录

    • bin 可执行程序
    • conf 配置
    • lib jar包
    • logs 日志
    • temp 临时文件
    • webapps 部署的web工程
    • work 运行时jsp翻译为Servlet的源码，和session钝化的目录
      • 钝化： 当服务器正常关闭时，还存活着的session (在设置时间内没有销毁) 会随着服务器的关闭被以文件 (“SESSIONS.ser”)的形式存储在tomcat 的work 目录下，这个过程叫做Session 的钝化。

  3. 启动Tomcat

    • 双击tomcat\bin\startup.bat，打开浏览器，访问http://localhost:8080和http://127.0.0.1:8080，测试是否安装成功

    • cmd中，进入tomcat\bin目录，运行catalina run命令

  4. 停止Tomcat

    • 关闭窗口
    • 在Tomcat窗口敲下CRTL+C
    • 双击tomcat\bin\shutdown.bat

  5. 修改Tomcat的端口号

    Tomcat默认端口号是8080，找到tomcat\config\server.xml找到Connector标签，修改port属性，重启Tomcat服务器生效

  6. 部署web工程到Tomcat服务器

    • 把web文件夹复制到tomcat\webapps目录下，访问localhost:8080/book/index.html即可

    • 进入tomcat\conf\Catalina\localhost，创建配置文件xxx.xml

  7. ROOT工程的访问/默认index.html页面的访问

    • http://ip:port/，没有工程名，默认访问ROOT工程
    • http://ip:port/工程名/，没有资源名。默认访问index.html

  8. IDEA整合Tomcat服务器

    File->Settings->Build->Application Servers->添加Tomcat Server

  9. IDEA中动态web工程的操作

    New->Module->Java Enterprise->选择Tomcat版本/在Additional Libraries and Frameworks下勾选Web Application/勾选Create web.xml/点击Next

  10. 动态web工程目录

    • src 自己编写的java源代码
    • web 用来存放web工程的资源文件，比如html/css/js文件
    • web\WEB-INF 此目录受服务器保护，浏览器无法直接访问到此目录下的内容
    • web\WEB-INF\web.xml 动态web工程的配置部署文件，可以配置web工程的组件，比如：Servlet组件/Filter过滤器/Listener监听器/Session超时
    • web\WEB-INF\lib 存放第三方的jar包（IDEA还需要自己配置导入）

  11. 在IDEA上部署工程到Tomcat上运行

  12. 配置资源热部署：修改On frame deactivation，热部署指的是修改文件以后，不用重启服务器就能更新页面
