jsp的三种语法

jsp头部的page指令

jsp的page指令可以修改jsp页面中的一些重要的属性或者行为

language 属性，表示jsp翻译后是什么语言文件，暂时只支持java

contentType属性，表示jsp返回的数据类型是什么，也是源码中response.setContentType()参数源码值

pageEncoding属性，表示当前jsp页面文件本身的字符集

import属性，用于导入包和类

errorPage属性，设置当jsp页面运行时出错，自动跳转去的错误信息页面

isErrorPage属性，设置当前页面是否是错误信息页面，默认为false

以下属性供out输出流使用

autoFlush属性，设置当out输出流缓冲区满了以后，是否自动刷新缓冲区，默认是true

buffer属性，设置缓冲区大小，默认8kb

session属性，设置访问当前jsp页面，是否会创建HttpSession对象。默认为true

extends属性，设置jsp翻译出来的java类默认继承哪个类

声明脚本的格式是：<%! 声明java代码 %>

作用：可以给jsp翻译出来的java类定义属性和方法甚至是静态代码块、内部类等

jsp中的九大内置对象

jsp的内置对象，是指Tomcat在翻译jsp页面成为Servlet源代码后，内部提供的对象

request

response

pageContext

session

application ServletContext对象

config ServletConfig对象

out jsp输出流对象

page 指向当前jsp的对象

exception 开启isErrorPage，异常对象

jsp的四大域对象

pageContext(PageContextImpl类)，当前jsp页面内有效

request(HttpServletRequest类)，一次请求内有效

session(HttpSession类)，一次会话范围内有效（打开浏览器访问服务器，直到关闭浏览器）

application(ServletContext类)，整个web工程内都有效（只要web工程不停止，都有效）

域对象可以像Map一样存取数据

使用优先顺序：pageContext/request/session/application

这样使用的原因，内存优化，当前jsp页面有效的数据，离开当前jsp页面，数据就被释放了

jsp中的out输出和response.getWriter输出的区别

当jsp页面中所有代码执行完成后会执行以下操作：

  1. 执行out.flush()操作，将out缓冲区中的数据追加写入response缓冲区末尾
  2. 会执行response的刷新操作，把全部数据写给客户端

底层源码使用out输出

out.write()和out.print()的区别：

  • out.write()输出字符串没问题，但是输出整型，会先将整型数据转化成字符类型(49->’1’)

  • out.print()输出任何数据，都是转换为字符串之后调用out.write()输出

统一使用out.print()

jsp静态包含(常用)

几十个jsp页面有一部分相同的内容，这些内容可以放进另一个jsp页面，供几十个jsp页面使用

静态包含不会翻译被包含的jsp页面

静态包含其实是把被包含的jsp页面的代码拷贝到包含的位置执行输出

main.jsp

footer.jsp

动态包含

page属性是指定你要包含的jsp页面的路径

  • 动态包含会把包含的jsp页面也翻译成java文件

  • 动态代理底层代码使用JspRuntimeLibrary.include(request,response,"/include/footer.jsp",out,false)去调用被包含的jsp文件，将request和out传递给被包含的jsp文件使用

  • 动态包含可以传递参数
